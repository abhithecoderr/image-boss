# Context Map: object-segmentation/worker.js

## 1. Purpose
Interactive object segmentation and extraction utilizing the Segment Anything Model (SlimSAM/SAM-2). Optimized for real-time feedback by separating the heavy feature encoding (Image Embeddings) from the lightweight point-prompted decoding.

## 2. Imports
- **@huggingface/transformers**:
  - `SamModel`/`Sam2Model`: High-precision segmentation graphs.
  - `AutoProcessor`: Handles the complex prompt-to-tensor mapping.
  - `RawImage`/`env`: Data structures and hardware control.

## 3. Dependencies
- **Uses**:
  - Hugging Face Hub (SlimSAM-77 uniform weights).
  - WebGPU (Primary Hardware Target).
- **Used by**:
  - [processor.js](file:///c:/projects/bg/my-ai-app/src/services/object-segmentation/processor.js) (IPC) - Coordinates user click-points and result rendering.

## 4. State Management

- **model/processor (Variables/Objects)**
  - **Syntax**: `let model = null; let processor = null;`
  - **Purpose**: Singleton references for the active SAM session to prevent redundant 100MB+ memory allocations.

- **cachedEmbeddings (Variable/Tensor)**
  - **Syntax**: `let cachedEmbeddings = null`
  - **Purpose**: High-dimensional feature map of the current image. Once generated by the "Encoder," it allows for instantaneous mask generation from new points.

- **cachedRawImage (Variable/RawImage)**
  - **Syntax**: `let cachedRawImage = null`
  - **Purpose**: Low-level image reference used for coordinate domain mapping.

## 5. Project Flow
1. **Intake**: Receives commands (`process`, `clear`) with an optional `ImageBitmap` and `points` array.
2. **Encoding Stage (Heavy)**:
   - On the first call for a new image, the **Heavy Encoder** runs to generate the `cachedEmbeddings`.
   - This pass is $O(ImageSize)$ and typically takes $1-3$ seconds on WebGPU.
3. **Prompt Mapping**: Normalized user click-coordinates (0.x, 0.y) are mapped back to the physical pixel dimensions of the AI's input domain.
4. **Decoding Stage (Fast)**:
   - The **Fast Decoder** runs using only the points and the cached embeddings.
   - This pass is $O(PointCount)$ and concludes in $<50$ms, enabling real-time "snapping" previews.
5. **Post-Processing**: The AI returns a "Mask Triplet" (Small, Medium, Large context options). These are thresholded and converted to Uint8 buffers.
6. **Export**: The mask options are returned to the main thread via transferables.

## 6. Code Structure

- **`clearCache` (Function)**
  - **Name (Type)**: clearCache (Cleanup)
  - **Syntax**: `function clearCache()`
  - **Purpose**: Manually invalidates tensors and image references to prevent memory accumulation.

- **`onmessage` (Handler)**
  - **Name (Type)**: onmessage (IPC Router)
  - **Syntax**: `self.onmessage = async ({ data }) => { ... }`
  - **Working**:
    - **Session Recovery**: Detects "Device Lost" or aborted WebGPU states and triggers a surgical session reload.
    - **Encoding Guard**: Checks if the image has changed; if not, skips directly to decoding (L123) for instant response.

- **Encoder Logic (Block)**
  - **Name (Type)**: Image Encoder (Stage)
  - **Syntax**: `cachedEmbeddings = await model.get_image_embeddings(lastImageInputs);`
  - **Working**: Implements the **SAM Feature Extraction Theory**. It creates a persistent latent representation of the image features.

- **Decoder Logic (Block)**
  - **Name (Type)**: Point Decoder (Stage)
  - **Syntax**: `const outputs = await model({ ...currentInputs, image_embeddings: cachedEmbeddings });`
  - **Working**: Uses the cached embedding "shortcuts" to generate the final mask pixels based on the provided positive/negative points.

- **`post_process_masks` (Function)**
  - **Name (Type)**: post_process_masks (Utility)
  - **Syntax**: `const masks = await processor.post_process_masks(predMasks, ...)`
  - **Working**: Scales the raw 256px AI masks back to the high-res input dimensions using bilinear interpolation.

## 7. Points To Consider
- **Decoder Persistence**: Consider maintaining the model state between clicks (L75) because the decoder's sub-50ms latency is dependent on stable memory allocation.
- **Coordinate Domain**: Note that points must be mapped using `cachedRawImage.width/height` (L76) to ensure the AI detects the correct features in the embedding space.
- **Mask Selection**: Consider passing all 3 SAM output masks to the UI (L77) so the user can resolve scale ambiguity (e.g., selecting an eye vs. the entire face).
- **WebGPU Recovery**: Note that long inference tasks can trigger "Device Lost" events on Windows; consider keeping the recovery logic (L78) robust.
