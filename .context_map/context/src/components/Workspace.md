# Context Map: Workspace.jsx


## 1. Purpose

The primary visual stage of the application. It manages the rendering of source and result images, provides the upload drop-zone, and orchestrates the various feature overlays (SAM, Mask Editor) that sit on top of the image results. It handles the bridge between React's state-driven lifecycle and the high-performance pixel rendering required for canvas operations.


## 2. Imports

- **React**:
  - Syntax: `import React, { useRef, useEffect } from 'react';`
  - Purpose: Provides core React functionality, including the `useRef` hook for direct DOM access and `useEffect` for side-effect synchronization.

- **useApp**:
  - Syntax: `import { useApp } from '../context/AppContext';`
  - Purpose: Accesses the global `AppContext` for image states (source/result), processing status, and the current active service.

- **useFileIngestion**:
  - Syntax: `import { useFileIngestion } from '../hooks/useFileIngestion';`
  - Purpose: Handles the logic for parsing and validating dropped or selected image files.

- **SAMOverlay**:
  - Syntax: `import SAMOverlay from './features/SAMOverlay';`
  - Purpose: Renders the interactive selection markers (positive/negative clicks) on top of the source image.

- **SegmentationCandidates**:
  - Syntax: `import SegmentationCandidates from './features/SegmentationCandidates';`
  - Purpose: Displays the secondary UI for selecting between multiple mask options generated by the SAM model.

- **ComparisonSlider**:
  - Syntax: `import ComparisonSlider from './features/ComparisonSlider';`
  - Purpose: Provides a high-performance before/after reveal slider, specifically used for upscaling workflows.

- **MaskEditorOverlay**:
  - Syntax: `import MaskEditorOverlay from './features/MaskEditorOverlay';`
  - Purpose: Renders the manual brush interface (canvas) for pixel-perfect mask refinement using Erase/Restore tools.


## 3. Dependencies

- **Uses**:
  - `useApp` for global state access.
  - `useFileIngestion` for file processing logic.
  - `canvas-utils.js` (dynamically imported) for result downloads.

- **Used by**:
  - `App.jsx` - As the central visual component in the main layout.


## 4. State Management

- **srcRef (Ref)**:
  - Syntax: `const srcRef = useRef(null);`
  - Purpose: Provides direct DOM access to the source canvas element for high-performance rendering of the original image buffer without triggering React reconciliation.

- **resRef (Ref)**:
  - Syntax: `const resRef = useRef(null);`
  - Purpose: Provides direct DOM access to the result canvas element for high-performance rendering of processed AI results and manual edits.

- **srcCanvasState (Variable/Object)**:
  - Syntax: `originalCanvas: srcCanvasState` (Destructured from `useApp`).
  - Purpose: The truth-source for the loaded image's pixel data, used to trigger the sync effect.

- **resCanvasState (Variable/Object)**:
  - Syntax: `resultCanvas: resCanvasState` (Destructured from `useApp`).
  - Purpose: The truth-source for the processed result image, used to trigger the sync effect for the result view.


### 1. Intake Initialization
- **Flow**: If `srcCanvasState` is null, the component renders a dedicated `upload-area`.
- **Files Involved**:
  - `useFileIngestion.js`: Captures the drop event and initiates the pixel-extraction pipeline.

### 2. Setup Phase
- **Flow**: Upon file ingestion, `srcCanvasState` is populated, triggering the dual-panel `preview-container`.
- **Files Involved**:
  - `AppContext.jsx`: Commits the original image and canvas buffers to the global state.

### 3. Synchronization Loop
- **Flow**: `useEffect` hooks monitor `srcCanvasState` and `resCanvasState` to manually sync pixels via `drawImage`.
- **Files Involved**:
  - `AppContext.jsx`: Provides the source/result buffers that trigger the sync effects.

### 4. Overlay Management
- **Flow**: Based on the `currentService`, overlays are mounted on top of the canvases.
- **Files Involved**:
  - `SAMOverlay.jsx`: Mounted for object-segmentation tasks.
  - `MaskEditorOverlay.jsx`: Mounted for manual mask refinement.

### 5. Synthesis/Download
- **Flow**: When `Download` is clicked, the browser-side file export is triggered.
- **Files Involved**:
  - `canvas-utils.js`: Orchestrates the `canvasToBlob` conversion and virtual link download.


## 6. Code Structure

- **Sync Source Canvas (Effect Hook)**:
  - Syntax: `useEffect(() => { ... }, [srcCanvasState]);`
  - Purpose: Synchronizes the original image state with the physical DOM canvas.
  - Working: Checks for `srcCanvasState` existence, resets the `srcRef` canvas dimensions to match the source exactly, and performs a 1:1 `drawImage` transfer onto the 2D context.

- **Sync Result Canvas (Effect Hook)**:
  - Syntax: `useEffect(() => { ... }, [resCanvasState]);`
  - Purpose: Synchronizes the AI result state with the physical DOM canvas.
  - Working: If `resCanvasState` exists, it updates dimensions and draws the buffer. Crucially, if `resCanvasState` is null, it invokes `ctx.clearRect` to ensure the UI doesn't display stale artifacts from previous sessions.

- **onDrop (Function)**:
  - Syntax: `const onDrop = (e) => { ... }`
  - Purpose: Handles native browser drop events.
  - Working: Prevents default browser behavior, extracts the first file from `dataTransfer.files`, validates the MIME type starts with 'image/', and forwards it to `handleFile`.


## 7. Points To Consider

- **The "Stale Result" Trap**: The `useEffect` for `resCanvasState` includes an explicit `else` branch for `ctx.clearRect`. This is critical because `resRef.current` persists across renders; without this, switching models within a service would leave the previous model's output visible until the next process completes.

- **Instance Validation**: The result sync logic includes an `isValidSource` check (`instanceof HTMLCanvasElement || OffscreenCanvas || ImageBitmap`) to prevent runtime crashes when handling mixed result types.

- **Reconciliation Bypass**: High-performance image data is never passed directly to `<canvas>` via React props. Instead, it is transferred via `drawImage` in effects to avoid the massive cost of React's shallow comparison on large pixel arrays.

- **Resolution Sync**: Canvas `width` and `height` must be updated on the DOM element *before* `drawImage` to ensure internal coordinate mappings match the source resolution exactly.
